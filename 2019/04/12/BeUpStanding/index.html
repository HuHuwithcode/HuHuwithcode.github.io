BeUpStanding

Posted on 2019-04-12 | Edited on 2019-04-14 | In Completed Projects | Comments: | Views:

[July 2018 - November 2018]
Course: Design Computing Studio 3 - Build

“BeAppStanding” is an iOS mobile application to further help the target users to sit less and stand more. This application and physical prototype were development by using native platforms (Swift,Xcode). The key component of my responsibility is to consider how to enhance the user experience and design the data visualisation by using emotion design and interface usability. In terms of it, the system can improve the cognition of target audience.

Introduction

BeAppStanding is the project initiated by Professor Genevieve Healy from the Public Health School at the University of Queensland. Professor Healy and her colleagues have created the BeUpStanding project in order to raise people’s awareness of sit less and stand more. This project, BeAppStanding, was created as the extension of the Be Up Standing project, the goal of this project is to create an iOS mobile application to further help the target users to sit less and stand more.

The solution for the project was to develop an iOS mobile application, the critical features of this application are “Personal Activity Goal” and “Team Activity Goal”. The application allows users to create the standing activity goal by setting how many times they want to stand up and the interval during a day, then the application will remind users to stand up and complete a set of activity by sending them a notification. In addition, users of the application are allowed to invite their friends or colleagues to form into an activity group and the team leader can create a team activity goal for each team member to complete, this feature was designed to encourage users to complete more activity goal.

The main target users of this application are office workers who spend plenty of time on sitting at office. Another target group for the application is the university students who also spend a huge amount of time on sitting for study. Nevertheless, this application is also targeting the people who constantly spend their time on sitting and have not enough exercise as this application aims to encourage people to stand more and be healthier. Since the main target users are the office workers, the main context of using the BeAppStanding application is during office hours when office workers spend most of time on sitting by the desk. The idea was to let the users to set up the activity goal after arriving in the office and starting to work, the application will start working and send push notification to users’ mobile phone as a reminder for standing.

Project Description

Application Overview

I designed and implemented the Medium-fidelity to High-fidelity prototype of the application independently using Adobe suit and conducted conceptual model based on user testing. Additionally, I learnt how to use the Swift to build an IOS application by using the Xcode.


By the end of this project, our team completed and provided a working application which can meet current requirements from product owner. The main features of the application have all been successfully deployed to the final product, main features including:

  1. Sign up and login
  2. Personal Activity Goal
  3. Team Activity Goal
  4. Activity Goal Data Visualisation for both personal and team– Pending Tree
  5. Account Management
  6. Award System

However, despite the full effort from our team and the application is meeting the major requirements of product owner, there are still some defects existed for the application due to the lack of development time. Firstly, the development team initially wanted the function which allows the user to login to the application with their third-party account, such as Google account and Facebook account, this function failed to implemented into the final product. Secondly, the function of setting activity goal failed to match the original design idea, initially the team wanted this function to allow user not just quickly set up an activity goal, but also customize their activity goal by editing more features such as start time and finish time of the activity goal, current function only allows a basic setting for the activity goal. Thirdly, although the team function is working and reaching its original design intention, users will not receive the notification of a team invitation, so currently they have no ability to reject the team invitation; users will not be invited to a team if they already enrolled in another team, but if they were invited to a team that they not intended to join, they have to manually sign out the team by accessing the management page.

Research/Design Assessment

In this project, the target is to design an iOS mobile application which is able to assist and encourage users with monitoring their daily stand up tasks by setting activity goal individually. Notification function which is one of the key designs of the BeAppStanding application to remind users of doing tasks in time. In addition, users are allowed perform team activity by forming a group. Team activity goal can be set by the team leader who creates the team. This unique design can greatly enhances user experience by allowing users to do activities with friends or colleagues. In the design process, we focus on convenience and interaction. All the main functions are deigned to put in ToDo page altogether which is easy to maintain and viewed by users. Design of Done page enables users to view their daily activity within 12 month calendar. Initially, only white, blue and red colours are used in the user interface because we considered that office workers prefer clearer and simpler visual feelings than complex design. However as the colour purple has been considered strongly related to the image of UQ, thus the UI design was re-designed so that users can see purple elements in some areas which aims to fit with UQ style. shows the changes of UI design.

I got the feedback of adding gamification elements to the data visualization in our application after first critique. We recognized that the advice is very valuable for the application so that after confirming with our client, we decided to use animated falling leaves from tree to replace the visualised data in the application. This tree was named ‘Pending Tree’. Animated falling leaves from tree enable users to interact with data more directly with better understanding of operational standing activities and users’ own completion status. It changes the previous design of visualised data that which was not interactive enough. Previous design is shown in below.

The tree will initially have 9 leaves, and keep current state if user keep finishing all the tasks every day. If user did not complete all the tasks in a day, tree will lose leaves. The leaves will grow back when user finish all the set plan in three days. The core design of this part improved the conventional sport Apps which are in lack of amusement elements, difficult to maintain and these functions motivate users to do activities continuously. In the design process, we selected big palm tree with green leaves as the symbol image. Palm tree grows in warm and coastal places which suits the image of Queensland’s warm and sunny weather. Green always represent refreshment which is the core concept of this application. Animated design was used to model the real tree, if user did not give them enough nutrition and water the leaves will fall. I applied this concept to our application that if user did not complete the tasks, leaves will fall however if user finish the tasks (give them enough nutrition and water) the leaves will growing back.

Icon of our application is also one of the important design parts. I redesign twice during the development process. Concepts like high-recognizable, high-resolution, visual consistent are applied in our deign. In the previous design we choose to use green or orange for the icon however I consider that purple is more fitted with UQ style, thus ultimate UQ’s symbol colour purple was selected to fill the most parts of the icon.
