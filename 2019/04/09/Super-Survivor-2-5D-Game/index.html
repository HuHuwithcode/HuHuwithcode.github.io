Introduction

“Super Survivor Game” is a small map adventure game of website. It was built by using the The HTML5 Creation Engine(PixiJS) and involve some form of physical interaction by using the MakeyMakey. Using the statement provided to my game by the Game Generator at http://orteil.dashnet.org/gamegen.

The protagonist of the game is a different kind of small box, the player needs to manipulate the red small box to move and avoid multiple small black boxes of attack. At the same time, players need to control the small red box to find a symbol of the “survival package” of the yellow box, and carry a small yellow box to move. Within the specified time, when the player places the yellow box in the correct position of the map and collides with the small black box, the movement of the small black box group can be stopped immediately. Finally, the player wins the game.

Prototype 2

I created an interactive prototype of my application interface that considers the key user functions, tasks and/or interactions. I was aiming to implement a complete or polished interface. However, functionality should be completed to a level that allows for clear understanding
of the elements being evaluated. Thus, when I was deciding on the aspect to be prototyped, consider what it is I want to know about how people would engage/interact with the game interface. According to this, I also created multiple prototypes to evaluate variations in functionality, task flow and interaction methods. As this is an early prototype, the design of the interface used placeholder and wireframe graphics to represent interface elements.


Prototype 3

I developed a prototype to evaluate the intended physical (non-digital) inputs/outputs of my game application. The key issues what I thinking about is the physical interactions should be appropriate to the game concept. And it should be prototyped to a standard that allows for useful evaluation. Another problem I need to address - which is the intended physical interaction of my game. In this process, I want to test the physical interaction/input method separate to the interface, and I also want to test how that physical input is represented back to the user through the interface.


Prototype 4

In the last step, I did the Dark-horse prototype. Because I felt that my concept is suffering from a lack of inspiration. Maybe it can provided some good idea for me to refine my game according to the feedback.
